MÓDULO 01 / 08 Iniciante
Fundamentos do 3D na Web
Antes de escrever uma linha de Three.js, é essencial entender por que a tecnologia existe, o que acontece dentro do browser quando uma cena 3D é renderizada e como preparar o ambiente para começar.
O que é WebGL?
WebGL (Web Graphics Library) é uma API JavaScript baseada no OpenGL ES 2.0/3.0 que permite renderizar gráficos 2D e 3D diretamente no browser sem plugins. Ela expõe a GPU do usuário ao JavaScript, permitindo operações massivamente paralelas.
O problema: WebGL puro é extremamente verboso. Criar um triângulo colorido exige dezenas de linhas de código, gerenciamento manual de buffers, shaders e estados da GPU. É aí que entra o Three.js.
WebGL puro
API de baixo nível. Controle total, mas muito código boilerplate. Usado quando se precisa de máxima performance ou funcionalidades não suportadas por frameworks.
Three.js
Abstração sobre WebGL. Oferece conceitos de alto nível (cena, câmera, luz, material) mantendo flexibilidade para personalização avançada via shaders.
Babylon.js
Alternativa mais "batteries-included" com física e sistema de nós integrados. Curva de aprendizado maior, mais voltada a jogos.
React Three Fiber
Three.js declarativo via JSX + React. Excelente para aplicações React, mas adiciona abstração sobre abstração.
Como a renderização funciona
O pipeline de renderização é a sequência de etapas que transforma dados 3D em pixels na tela. Entender esse pipeline é fundamental para debugar problemas visuais e otimizar performance.
1. CPU → GPU
Vértices, índices e uniforms são enviados da memória RAM para a VRAM via buffers.
2. Vertex Shader
Programa GLSL executado para cada vértice. Calcula a posição final na tela (gl_Position).
3. Rasterização
A GPU converte triângulos em fragmentos (pixels potenciais) — processo automático.
4. Fragment Shader
Programa GLSL para cada fragmento. Define a cor final do pixel (gl_FragColor).
⚠
Atenção: A GPU executa os shaders em paralelo para milhões de pixels simultaneamente. Por isso, otimizações no fragment shader têm impacto enorme na performance.

MÓDULO 02 / 08 Iniciante
Cena, Geometrias e Materiais
Estes são os três pilares de qualquer cena Three.js. Compreender como Scene, Camera, Renderer, Geometrias e Materiais interagem é a base para tudo que vem depois.
O trio fundamental: Scene, Camera, Renderer
Pense na metáfora de um set de filmagem: a Scene é o estúdio com todos os objetos e atores; a Camera é o equipamento de filmagem que define o que aparece no quadro; o Renderer é o projetor que exibe o resultado final.
Scene
Grafo de cena (scene graph) que armazena todos os objetos, câmeras e luzes. Os objetos são organizados em hierarquia pai-filho via add().
PerspectiveCamera
Simula perspectiva humana. Parâmetros: FOV (campo de visão em graus), aspect ratio, near (mínimo de renderização) e far (máximo de renderização).
OrthographicCamera
Sem perspectiva — objetos distantes têm o mesmo tamanho que próximos. Útil para minimapas, jogos 2D e visualizações técnicas.
WebGLRenderer
Renderiza a cena usando WebGL. Configure antialias: true e setPixelRatio() para qualidade em telas de alta resolução.
Geometrias
Uma Geometry define a forma de um objeto — seus vértices, arestas e faces. Three.js oferece geometrias primitivas prontas e permite criar geometrias customizadas via BufferGeometry.
BoxGeometry
Cubo. Parâmetros: width, height, depth, widthSegments, heightSegments, depthSegments.
SphereGeometry
Esfera. Parâmetros: radius, widthSegments, heightSegments. Mais segmentos = mais suave.
PlaneGeometry
Plano 2D. Útil para chãos, paredes e superfícies planas com texturas.
CylinderGeometry
Cilindro/cone. Pode ter topo e base com raios diferentes, criando troncos de cone.
TorusGeometry
Rosquinha. Parâmetros: radius (raio do torus), tube (raio do tubo), radialSegments, tubularSegments.
BufferGeometry
Geometria customizada. Define vértices manualmente via Float32Array para formas arbitrárias.
◆
Dica de performance: Geometrias são armazenadas na GPU. Reutilize a mesma instância de geometria em múltiplos objetos — não crie uma nova geometria para cada mesh. Use InstancedMesh quando tiver muitos objetos idênticos (veja Módulo 08).
Materiais
O material define a aparência visual da geometria — cor, textura, como ele reage à luz. A escolha do material impacta diretamente a performance.
MeshBasicMaterial
Não reage à luz. Sempre a mesma cor independente da iluminação. O mais performático. Bom para wireframes e debug.
MeshStandardMaterial
PBR (Physically Based Rendering). Parâmetros: roughness (rugosidade) e metalness (metalicidade). O padrão para cenas realistas.
MeshPhongMaterial
Modelo de iluminação Phong (mais antigo que PBR). Mais rápido que Standard, mas menos preciso fisicamente.
ShaderMaterial
Usa shaders GLSL customizados. Máximo controle, mas exige conhecimento de GLSL (Módulo 07).
Transformações: position, rotation, scale
Todo objeto 3D no Three.js herda de Object3D, que fornece as propriedades de transformação. Estas formam a TRS Matrix (Translation, Rotation, Scale).
Transformações de objetos
// Position — Vector3 mesh.position.set(1, 2, 0) // ou mesh.position.x = 1 // Rotation — Euler (radianos!) mesh.rotation.y = Math.PI / 4 // 45 graus // Scale mesh.scale.set(2, 0.5, 2) // dobro na X e Z, metade na Y // lookAt — aponta para uma posição no espaço mesh.lookAt(new THREE.Vector3(0, 0, 0)) // Group — agrupa objetos para transformá-los juntos const group = new THREE.Group() group.add(mesh1) group.add(mesh2) scene.add(group) group.rotation.y = Math.PI // rotaciona os dois juntos

MÓDULO 03 / 08Iniciante
Iluminação e Texturas
Iluminação e texturas são o que transforma uma cena técnica em algo visualmente convincente. Este módulo explora o PBR (Physically Based Rendering) e como usá-lo para criar materiais realistas.
Tipos de luz
Cada tipo de luz tem um comportamento diferente e um custo de performance diferente. Use o mínimo necessário para o efeito desejado.
AmbientLight
Iluminação global uniforme. Não cria sombras. Simula luz refletida pelo ambiente. Sempre use junto com outras luzes para evitar sombras totalmente pretas.
DirectionalLight
Raios paralelos, como o sol. Alta performance. Cria sombras nítidas. Posição define a direção, não a distância.
PointLight
Irradia em todas as direções de um ponto, como uma lâmpada. Tem atenuação por distância (decay). Mais custosa que DirectionalLight.
SpotLight
Cone de luz. Parâmetros: angle, penumbra, distance. A mais versátil mas também a mais custosa.
⚠
Performance: Sombras (shadow maps) são caras. Limite a 1-2 luzes com castShadow: true. Configure shadow.mapSize com valores potências de 2 (512, 1024, 2048).
PBR e MeshStandardMaterial
PBR (Physically Based Rendering) é um conjunto de técnicas que simula como a luz interage com superfícies reais. O MeshStandardMaterial implementa PBR com dois parâmetros chave:
roughness
0 = espelho perfeito. 1 = completamente difuso (madeira fosca, pedra). Controla a dispersão da luz refletida.
metalness
0 = não-metal (plástico, madeira). 1 = metal puro (aço, ouro). Afeta como as reflexões se comportam.
map
Textura de cor base (albedo). Multiplica pela cor do material.
normalMap
Simula irregularidades de superfície sem adicionar geometria. Essencial para realismo.
TextureLoader e PBR
const loader = new THREE.TextureLoader() const material = new THREE.MeshStandardMaterial({ map: loader.load('/textures/color.jpg'), normalMap: loader.load('/textures/normal.jpg'), roughnessMap: loader.load('/textures/roughness.jpg'), metalnessMap: loader.load('/textures/metalness.jpg'), aoMap: loader.load('/textures/ao.jpg'), // ambient occlusion roughness: 0.5, // valor base (multiplicado pelo mapa) metalness: 0.0, }) // Environment Map para reflexões import { RGBELoader } from 'three/examples/jsm/loaders/RGBELoader.js' new RGBELoader().load('/env.hdr', (texture) => { texture.mapping = THREE.EquirectangularReflectionMapping scene.environment = texture // afeta todos os materiais PBR scene.background = texture // opcional: usar como fundo })

    Animação e Interatividade
    Movimento e resposta ao usuário são o que diferencia uma cena 3D de uma imagem estática. Este módulo explora o loop de animação, raycasting e técnicas de câmera dinâmica.
    O loop de animação
    O requestAnimationFrame sincroniza com o refresh da tela (tipicamente 60fps). Para animações frame-rate independentes, use o Clock do Three.js para medir o tempo decorrido.
    Loop de animação robusto
    const clock = new THREE.Clock() function animate() { requestAnimationFrame(animate) const delta = clock.getDelta() // segundos desde o último frame const elapsed = clock.getElapsedTime() // segundos desde o início // Frame-rate independente: velocidade constante a qualquer fps mesh.rotation.y += 1.0 * delta // 1 radiano por segundo // Animação baseada em tempo (sin/cos para movimento cíclico) mesh.position.y = Math.sin(elapsed * 2) * 0.5 renderer.render(scene, camera) } animate()
    Raycasting — interação com objetos 3D
    O Raycaster projeta um raio da câmera para o espaço 3D na direção do cursor do mouse e detecta quais objetos esse raio intersecta. É a base para hover, clique e seleção em cenas 3D.
    Raycasting com hover e clique
    const raycaster = new THREE.Raycaster() const pointer = new THREE.Vector2() // Converter coordenadas do mouse para espaço normalizado (-1 a 1) window.addEventListener('mousemove', (e) => { pointer.x = (e.clientX / window.innerWidth) * 2 - 1 pointer.y = -(e.clientY / window.innerHeight) * 2 + 1 }) // No loop de animação: raycaster.setFromCamera(pointer, camera) const hits = raycaster.intersectObjects(scene.children) if (hits.length > 0) { const obj = hits[0].object obj.material.color.set('#FF6B35') // hover: muda cor }

    Modelos 3D e Animações
    Importar assets profissionais criados em Blender, Maya ou outras ferramentas é o passo que une o mundo do design 3D com o código. O formato GLTF é o padrão da web 3D.
    Formatos 3D e por que GLTF
    O GLTF (GL Transmission Format) é o "JPEG do 3D" — otimizado para transmissão e carregamento rápido no browser. O formato GLB é a versão binária compacta do GLTF (tudo em um arquivo).
    GLTF / GLB
    Padrão moderno. Suporta PBR, animações, morfologia, câmeras e muito mais. Use sempre que possível.
    OBJ
    Formato legado simples. Não suporta animações. Útil para geometrias estáticas simples.
    FBX
    Formato proprietário da Autodesk. Suporta animações mas é pesado. Evite se possível — converta para GLTF.
    Carregando e animando um modelo GLTF
    import { GLTFLoader } from 'three/examples/jsm/loaders/GLTFLoader.js' let mixer const loader = new GLTFLoader() loader.load('/models/character.glb', (gltf) => { const model = gltf.scene model.scale.setScalar(0.5) scene.add(model) // Configurar AnimationMixer mixer = new THREE.AnimationMixer(model) // Reproduzir primeira animação em loop const action = mixer.clipAction(gltf.animations[0]) action.play() // Crossfade entre animações const idle = mixer.clipAction(gltf.animations[0]) const walk = mixer.clipAction(gltf.animations[1]) idle.play() // Para trocar: idle.crossFadeTo(walk, 0.5, true) }) // No loop de animação — SEMPRE atualizar o mixer function animate() { requestAnimationFrame(animate) const delta = clock.getDelta() if (mixer) mixer.update(delta) renderer.render(scene, camera) }

    Física com Cannon.js
    Simulação física real: gravidade, colisões e reações físicas. Cannon-es é a versão moderna e mantida do cannon.js, compatível com ES Modules.
    Como a física funciona com Three.js
    Three.js cuida da renderização. Cannon.js cuida da física. Você mantém dois mundos em paralelo: o mundo visual (Three.js) e o mundo físico (Cannon-es). A cada frame, você copia as posições e rotações do mundo físico para os objetos visuais.
    World
    O mundo físico. Contém gravidade, broadphase e a lista de bodies. Chamar world.step() avança a simulação.
    Body
    Um objeto físico. Tem massa, velocidade, força e posição. mass: 0 = estático (imóvel).
    Shape
    A forma geométrica para colisão: Box, Sphere, Plane, Cylinder, ConvexPolyhedron.
    ContactMaterial
    Define como dois materiais interagem: fricção (friction) e restituição/bouncing (restitution).
    Setup completo de física
    import * as CANNON from 'cannon-es' // Mundo físico const world = new CANNON.World() world.gravity.set(0, -9.82, 0) // gravidade realista world.broadphase = new CANNON.NaiveBroadphase() // Corpo físico — esfera quicando const sphereShape = new CANNON.Sphere(0.5) const sphereBody = new CANNON.Body({ mass: 1 }) sphereBody.addShape(sphereShape) sphereBody.position.set(0, 5, 0) world.addBody(sphereBody) // Chão estático (mass: 0 = imóvel) const groundBody = new CANNON.Body({ mass: 0 }) groundBody.addShape(new CANNON.Plane()) groundBody.quaternion.setFromAxisAngle(new CANNON.Vec3(1,0,0), -Math.PI/2) world.addBody(groundBody) // No loop — sincronizar física com Three.js function animate() { requestAnimationFrame(animate) world.step(1/60, delta, 3) // fixedStep, deltaTime, maxSubSteps // Copiar posição e rotação da física para o mesh sphereMesh.position.copy(sphereBody.position) sphereMesh.quaternion.copy(sphereBody.quaternion) renderer.render(scene, camera) }

    Shaders e GLSL
    Shaders são programas que rodam diretamente na GPU. Dominar GLSL abre um nível completamente novo de controle criativo — efeitos impossíveis de obter com os materiais nativos do Three.js.
    GLSL: a linguagem dos shaders
    GLSL (OpenGL Shading Language) é uma linguagem com sintaxe similar ao C. É executada em paralelo na GPU — cada vertex shader roda para cada vértice, e cada fragment shader roda para cada pixel, simultaneamente.
    Tipos base
    float, int, bool, vec2, vec3, vec4, mat2, mat3, mat4. Sem conversão implícita — use float(intVar).
    Uniforms
    Valores passados do JavaScript para o shader. Iguais para todos os vértices/fragmentos no mesmo draw call.
    Attributes
    Valores por vértice (só no vertex shader): position, normal, uv. Lidos da BufferGeometry.
    Varyings
    Valores passados do vertex para o fragment shader. Interpolados automaticamente entre vértices.
    ShaderMaterial completo — onda animada
    const material = new THREE.ShaderMaterial({ uniforms: { uTime: { value: 0 }, uColor: { value: new THREE.Color('#3DFFE4') } }, vertexShader: ` uniform float uTime; varying vec2 vUv; void main() { vUv = uv; vec3 pos = position; pos.z += sin(pos.x * 3.0 + uTime * 2.0) * 0.2; gl_Position = projectionMatrix * modelViewMatrix * vec4(pos, 1.0); } `, fragmentShader: ` uniform vec3 uColor; varying vec2 vUv; void main() { float brightness = vUv.y; // mais claro no topo gl_FragColor = vec4(uColor * brightness, 1.0); } ` }) // Atualizar uTime no loop material.uniforms.uTime.value = clock.getElapsedTime()
    Post-processing com EffectComposer
    Post-processing aplica efeitos sobre a imagem renderizada — como filtros de foto, mas em tempo real. O EffectComposer encadeia múltiplos "passes" de processamento.
    ◈
    UnrealBloomPass seletivo: Use layers para controlar quais objetos emitem bloom. Objetos na layer 1 recebem bloom; os da layer 0 não. Combine dois renderers com addPass() para o efeito final.

    O último módulo trata de escalabilidade: como manter 60fps com cenas complexas, técnicas de otimização avançadas e como fazer deploy de uma aplicação Three.js em produção.
    InstancedMesh — o segredo dos 60fps
    Cada objeto na cena normalmente gera um "draw call" para a GPU. Muitos draw calls matam a performance. O InstancedMesh renderiza milhares de objetos idênticos em um único draw call, usando uma matriz de transformação por instância.
    InstancedMesh — 10.000 cubos com 1 draw call
    const COUNT = 10000 const geometry = new THREE.BoxGeometry(0.1, 0.1, 0.1) const material = new THREE.MeshStandardMaterial() const mesh = new THREE.InstancedMesh(geometry, material, COUNT) scene.add(mesh) // Posicionar cada instância com uma Matrix4 const matrix = new THREE.Matrix4() const position = new THREE.Vector3() const quaternion = new THREE.Quaternion() const scale = new THREE.Vector3(1, 1, 1) for (let i = 0; i < COUNT; i++) { position.set( (Math.random() - 0.5) * 20, (Math.random() - 0.5) * 20, (Math.random() - 0.5) * 20 ) matrix.compose(position, quaternion, scale) mesh.setMatrixAt(i, matrix) } mesh.instanceMatrix.needsUpdate = true
    Outras técnicas de performance
    LOD
    Level of Detail — exibe versões menos detalhadas do modelo conforme a distância da câmera aumenta. Reduz o número de triângulos renderizados.
    Frustum Culling
    Three.js automaticamente não renderiza objetos fora do campo de visão da câmera. Certifique-se que geometry.boundingSphere está calculado.
    Dispose
    Libere memória GPU ao remover objetos: geometry.dispose(), material.dispose(), texture.dispose(). Memory leaks são um problema real em Three.js.
    CSS2DRenderer
    Renderiza elementos HTML em world space, sincronizados com a câmera. Perfeito para labels e UI dentro da cena 3D.
